================================================================================
ARCHITECTURE HIGHLIGHTS
================================================================================

Design Pattern: Model-View-Controller (MVC)
  • Model: Bank core classes (unchanged)
  • View: QML interface (app/main.qml)
  • Controller: BankBridge (bank_bridge.cpp)

Signal Flow:
  User Action → QML Button → BankBridge Slot → Bank Operation
       ↓                                            ↓
  UI Update ← Signal Emission ← Operation Result

Data Flow:
  • Accounts stored in accounts.json (JSON format)
  • Automatically saved on "Save" button click
  • Automatically saved on application exit
  • Fully compatible with original CLI version
